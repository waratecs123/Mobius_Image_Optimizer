## Security Policy

### Supported Versions

Security updates are provided for the latest version of the Möbius platform and its associated applications. Older versions may not receive security patches.

### Reporting a Vulnerability

If you discover a security vulnerability in the Möbius platform or any of its components, please report it responsibly. Do not disclose the issue publicly until it has been addressed.

To report a vulnerability:  
- Send an email to the repository maintainer or open a private issue on GitHub if supported.  
- Provide a detailed description of the vulnerability, including steps to reproduce, affected components, and potential impact.  
- Include any relevant code snippets or proof-of-concept exploits.

The maintainer will acknowledge receipt within 48 hours and aim to provide a fix or mitigation within 14 days, depending on the severity. Updates on the status will be communicated via the reporting channel.

### Vulnerability Disclosure

Once a vulnerability is resolved, details may be disclosed publicly after a reasonable embargo period to allow users to update. Credit will be given to the reporter unless anonymity is requested.
